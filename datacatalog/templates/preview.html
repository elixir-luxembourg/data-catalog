Usage Rules

Do and Don't Examples

Each pair shows one rule. blue-900 for primary, blue-950 for dark fills, red-900 only for destructive or attention, sky-* for LCSB Centre surfaces.

Buttons and destructive actions

Primary fills with blue-900, secondary outlines in blue-900, destructive outlines in red-900. Red is never the primary CTA.

Do
Don't

Don't promote routine actions to red, and don't use sky as a small button surface with white text.

Links and active navigation

Inline links and active nav use blue-900. Red links are reserved for destructive or attention destinations.

Do
Overview Datasets Requests Delete account

Inline body link: read the submission guide.

Don't
Overview Datasets Requests

Don't mark routine active state in red, and don't use sky for small navigation labels.

Forms and validation

Inputs use neutral borders. Required markers and error messages use red-900. Focus rings use blue-900.

Do

A title is required.

Don't

This error message is too easy to miss.

Alerts and feedback

Neutral and informational feedback stays in blue or sky. Red is reserved for failure states.
